# 변경 이력 (Changelog)

> **최종 수정**: 2025-12-22
> **관련 문서**: [dev-quickstart.md](../dev-quickstart.md)

## 개요

- **목적**: 프로젝트 변경 이력을 체계적으로 관리
- **구조**: 메뉴/기능별 파일 + Phase별 상세 문서
- **규칙**: 새 변경사항은 해당 메뉴의 changelog 파일에 추가

---

## 파일 목록

### 메뉴별 변경 이력

| 파일 | 대상 | 설명 |
|-----|------|------|
| `character.md` | 캐릭터 검색 | 캐릭터 검색, 프로필, 원정대, 통합 메뉴 |
| `auction.md` | 경매장/거래소 | 거래소 검색, 통계, 그래프 |
| `reforge.md` | 재련 | 일반/상급 재련, 뭉가, 음돌 계산기 |
| `raid.md` | 레이드 | 레이드 파티 관리 |
| `admin.md` | 관리자 | 통계 수집, 레이드 카탈로그 |
| `common.md` | 공통 | 라우팅, 레이아웃, 공통 컴포넌트 |
| `backend.md` | 백엔드 | API, 서비스, 스케줄러 |

## 작성 규칙

### 날짜 형식
```markdown
### YYYY-MM-DD: 변경 제목
```

### 항목 구조
```markdown
### 2025-12-21: 기능 추가/수정 제목

**변경 유형**: Feature | Fix | Refactor | Performance

**변경 내용**:
- 변경 사항 1
- 변경 사항 2

**영향받은 파일**:
- `파일경로1`
- `파일경로2`

```

## AI 세션 가이드

### 변경사항 기록 시
1. 해당 메뉴의 changelog 파일에 추가
2. 여러 메뉴에 걸친 변경은 각 파일에 기록
3. 공통 변경(라우팅, 레이아웃 등)은 `common.md`에 기록
4. 백엔드 변경은 `backend.md`에 기록

### 새 메뉴 추가 시
1. `docs/changelog/` 폴더에 새 파일 생성 (예: `newmenu.md`)
2. 이 README의 파일 목록 업데이트
3. `docs/README.md` 업데이트
